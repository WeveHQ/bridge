# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in Weve Bridge, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **engineering@weve.security** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 2 business days and aim to provide a fix or mitigation timeline within 5 business days.
